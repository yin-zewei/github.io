BootStrap框架

发表于 2020-06-11 | 分类于 javaweb | 阅读次数
字数统计 339 | 阅读时长 1
  • 概念:前端开发框架，开发人员可在框架基础上进一步开发
  • 好处
    • 定义了很多css样式和js插件，得到丰富的页面效果
    • 响应式布局，兼容不同分辨率的设备
  • 快速入门
    • 下载bootstrap
    • 项目中引入，复制到项目当前目录
    • 创建html页面，引入必要的资源文件
  • 响应式布局
    • 同一套页面可以兼容不同分辨率的设备
    • 实现:依赖于栅格系统，将一行分成12个格子,然后可以让元素占得格子数量进行改变
    • 步骤
      • 定义容器，相当于table
        • container，比起container-fluid，会有留白
        • container-fluid：每种设备都是100%宽度
      • 定义行，想当于tr，row
      • 定义元素，指定该元素在不同的设备上所占的各自数目。样式：col-设备代号-格子数目
        • 设备代号
          • xs：超小屏幕 手机 <768px 例：col-xs-12
          • sm:小屏幕 平板 >=768px
          • md:桌面显示器 >=992px
          • lg:大屏幕 >=1200px
      • 注意
        • 一行如果格子数目超过12，则超出部分自动换行
        • 栅格类属性可以向上兼容
        • 如果设备宽度小于设备代号
  • CSS样式和js插件
    • 全局css样式
      • 按钮
      • 图片
      • 表格
      • 表单
    • 组件
      • 导航条
      • 分页条
    • 轮播图

基本概念

  • JavaWeb:使用Java语言开发基于互联网的网页
  • 软件架构
    • C/S结构:客户端/服务器架构
    • B/S架构:浏览器/服务器架构
  • 资源
    • 静态资源:文字，图片，视频等资源，HTML，CSS，JS语言去实现
    • 动态资源:动态网页发布的资源，如jsp/servlet，服务器会把动态资源转为静态资源再发到浏览器。

Html

  • 概念:Hyper Text Markup Language超文本标记语言
  • Notes:标记语言不是编程语言，没有逻辑性，只需要把这个标签打上去就行。
  • 语法
    1. html文件后缀:html，htm
    2. 标签分为
      • 围堵标签，由开始标签和闭合标签组成，如< head >< /head >，可以嵌套
      • 自闭合标签，开始标签和闭合标签为一体，如< br/ >
    3. 开始标签中可以定义属性，属性由键值对构成，需要用引号引起来(单引双引都可以)
  • 标签
    • 文件标签:构成html最基本的内容
      • html:根标签
      • head:头标签。指定html文档的属性，引入外部的资源
      • title:标题标签
      • body:体标签
      • < !DOCTYPE html>:h5定义该文档为html文档
    • 文本标签:
      • < !— — >:注释标签
      • h1-h6:标题标签
      • p:段落标签
      • br:换行标签
      • hr:显示水平线
      • b:字体加粗
      • i:字体斜体
    • 图片标签:
      • img
        • src:图片路径
        • 绝对路径(//)
        • 相对路径(./src)，以点开头
    • 列表标签
      • li:列表项，封装在列表其中
      • ol:有序列表
      • ul:无序列表
    • 链接标签
      • a:标签
        • href:资源网址，可以是url，也可以是本地的资源，打开客户端等
        • target:url是否在本页面打开
    • 快标签
      • 把代码包裹出来，便于css进行样式选择
      • span:行内标签
      • div:块级标签
    • 语义化标签
      • 概念:提高程序可读性
      • header:页眉
      • footer:页脚
    • 表格标签
      • table标签包裹
      • tr:行标签
      • td:单元格标签
      • th:表头单元格
      • caption:表格标题
      • thead:表格头
      • tbody:表格体
      • tfoot:表格脚
    • 表单标签
      • 概念:用于采集用户输入的数据，用于和服务器交互
      • form标签:定义一个输入的范围，提高可读性
        • input:输入标签
          • type属性:
            • text:文本输入框
            • password:密码输入框
            • radio:单选框
              • 注意:要使多个单选框实现单选的效果，那么必须使得name属性值相同
              • 如果要使得能提交到服务器，那么必须指定value的值
            • checkbox:复选框
              • 同单选框的注意事项，但是可以实现多选
            • file:文件提交框，用于提交文件
            • hidden:隐藏域，用于提交用户看不到的信息
            • checkcode:验证码
            • submit:提交框
              • 提交表单数据到服务器
            • image:图片提交按钮，点击会提交表单
          • placeholder:指定输入框的提示文字
          • lable:指定输入项的文字提示，for属性可以和后面的input标签的id属性对应，进而点击会进入input框
        • select:下拉列表
          • option:指定下拉列表的值
          • 提交成功的前提是select的name不为空，option的value不为空
        • textvalue:文本域
          • rows，cols:限制多少行和列
      • 属性:action，method
      • GET与POST请求的区别:GET会直接将请求封装到请求行中，POST会将请求加入到请求体中，GET对请求长度有要求，POST无要求，GET不安全，POST较安全
  • 静态html开发流程
    • 观察布局，使用table
    • 如果某一行一个单元格，就一个tr+td
    • 如果多个单元格，那么嵌套一个表格tr+table+td

ECMAscript

      • 外部js:通过script标签和src引入

    • 数据类型

      • 原始数据类型:
        • number:数字。整数、小数、NaN
        • string:字符串。单引或双引，没有字符的概念
        • boolean:true和false
        • null:一个对象为空的占位符
        • undefined:未定义。一个变量魏碑初始化类型
      • 引用数据类型:即为对象

      • 逻辑运算符:&&，||，!

        • null和undefined为false
      • 三元运算符:，?，:

      • js中可以不写分号，如果只有一行的话

这篇文章将告诉你如何配置git，以及学完这篇文章后，你将可以将本地的代码提交到GitHub，并且从GitHub下载代码，以及更新，回溯代码。

获得一个git仓库

有两种方法

  • 本地初始化一个仓库
    • cd project
    • git init
  • 从GitHub clone一个仓库
    • git clone url
      无论哪一种，最后都可以获得一个仓库

提交代码

  • git add *.py
  • git commit -a 本地提交所有修改
  • git push < name >
    • 之前要设置git remote add < name > < url >
