倍增LCA

发表于 2020-05-03 | 分类于 算法 | 阅读次数
字数统计 428 | 阅读时长 2

LCA是指求最近公共祖先的算法，如果使用暴力，那就是从下至上，但肯定会爆RE
使用倍增优化后，复杂度nlogn
步骤：

  • 初始化
    • 数据输入
    • lg优化(初始化lg[i]=log2(i)+1)
    • dfs初始化fa[i][j]=i节点的2^j祖先
  • LCA
    • 将a,b的深度调为一致
    • 如果相同，返回该祖先
    • 如果不同
      • i = lg(a)-1 -> 0,i 为 倍增的倍数
      • 如果fa[a][i] != fa[b][i]向上调节，逐渐到LCA的儿子节点

最小生成树

发表于 2020-05-02 | 分类于 算法 | 阅读次数
字数统计 647 | 阅读时长 3

以下介绍两种：Prim和Kruskal

Prim
Prim适用于稠密图，堆优化后复杂度VlogV
主要步骤：

* 随意取一条边
* 将这条边相邻的边加进来
* 取一个最小的边
* 直到有n-1条边

Kruskal
Kruskal适用于稀疏图
主要步骤：

* 将边排序
* 取最短边
* 查看是否连通
* 不连通就加进来，连通就跳过
* 继续取最短边，直到达到n - 1条

大组合数取模

发表于 2020-04-21 | 分类于 算法 | 阅读次数
字数统计 313 | 阅读时长 1

一般的组合数可以使用常规的递归得到，但是大组合数取模的复杂度太大，可以使用lucas定理进行优化;
原题
假设组合数函数为C(n,m) p为模数,fac(n)为阶乘函数
那么$C_n^m\%p = C_{n/p}^{m/p}C_{n\%p}^{m\%p}\%p$
其中，第一项接着lucas递归，第二项直接求解
ie.直接求解时可以使用逆元运算，避免除数，即$C_n^m=n!m^{p-2}*{(n-m)}^{p-2}\%p$

#include<bits/stdc++.h>
#define ll long long
const ll maxn = 1e5+5;
using namespace std;
ll f[maxn],p;
ll lucas(ll n,ll m);
ll C(ll n,ll m);
ll qpow(ll n,ll m);
int main(){
    //freopen("in.txt","r",stdin);
    ll T,a,b;
    cin>>T;
    while(T--){
        cin>>a>>b>>p;
        f[0]=1;
        for(ll i = 1;i <= p;++i)f[i]=i*f[i-1]%p;
        cout<<lucas(a+b,a)<<endl;
    }
    return 0;
}
ll qpow(ll n,ll m){
    ll ans = 1;
    for(int i = m;i;i>>=1,n=n*n%p){
        if(i&1)ans = ans * n % p;
    }
    return ans;
};
ll C(ll n,ll m){
    if(m>n)return 0;
    return f[n]*qpow(f[m],p-2)%p*qpow(f[n-m],p-2)%p;
};
ll lucas(ll n,ll m){
    if(m==0)return 1;
    return lucas(n/p,m/p)*C(n%p,m%p)%p;
};
